# 발주 자동화 — 현황 정리
작성일: 2026-04-17

---

## 우리가 만든 것 — 기능 개요

### 1. 원본 발주서 제작기 (`/`)
신규 제품을 처음 등록할 때 쓰는 화면.

- 브랜드·시즌구분·카테고리1/2·제품명·패키지원가·부자재원가·리드타임 입력
- 제품 행마다 item코드·model번호·MOQ·상품명·바코드·원가·수출가·사진 입력
- 포장 이미지, 외박스 이미지 업로드 (ILYON/누아트스튜디오는 기본이미지 자동 적용)
- 생성 시 **xlsx 다운로드 + NAS `통합발주서원본` 폴더에 자동 저장**
- 바코드 중복 확인: 입력 blur 시 구글시트 카테고리DB 조회 → 빨간 하이라이트
- 카테고리1/2 자동완성: 기존 DB 값 드롭다운 제공
- **생성과 동시에 구글시트 `카테고리 포함 db`에 행 자동 추가**
  - 아이템코드 HA-1 → HA-9999 → HB-1 자동 채번
  - 최종원가 = 제품원가 + 패키지원가 + 부자재원가 자동 계산

---

### 2. 발주 실행 (`/발주`)
기존 원본 발주서에서 수량을 채워 실제 발주서를 만드는 화면.

- 바코드 입력 시 **제품명으로 자동완성** (통합발주서원본 카탈로그 검색)
- 수량 + 사유(정기발주/신제품/수동발주 또는 직접입력) 입력
- 사유 일괄 적용 버튼 (모든 행에 동시 적용)
- 바코드+수량이 입력된 행이 하나 이상일 때만 **상품 조회 버튼 활성화**
- 조회 결과: 원본파일별 그룹카드, PO코드 제안(구글시트 최대번호+1), 원가·합계 표시
- 조회 후 DB 카드가 자동으로 열리며 **조회된 상품명+바코드만** 표시
- 발주 실행 시:
  - 원본 발주서 복사 → 해당 바코드 행에 수량 채움 → 나머지 행은 내용 비우고 행 높이 0 (이미지 유지)
  - 구글시트 **엔대시 발주이력** 기록
  - 구글시트 **중국발주요청리스트** 하단에 행 추가 (FA prefix → 일반수량열, 나머지 → 쿠팡수량열)
  - **이메일 자동 발송**: 여러 그룹도 1통으로 합쳐서 발송 (첨부파일 여러 개)

---

### 3. 재고현황판 (`/재고현황`)
매주 재고 상태를 한눈에 보는 분석 테이블.

- **부분 업로드 지원**: 파일 1개만 올려도 조회 가능. 안 올린 파일은 기존 저장 데이터 유지
- **구글시트 업데이트 버튼**: 파일 업로드 없이 구글시트만 다시 읽어 미입고·중국재고 갱신
- 7개 파일 업로드: 사방넷·basic·forecast·누아트sku·PO_SKU_LIST·주문서·재고현황표(선택)
- 구글시트 **중국발주요청리스트**에서 중국재고·미입고량 자동 조회
- 분석 결과를 `output/재고현황_최근.json`에 저장 → 다음 접속 시 파일 없이 로드
- 컬럼: 카테고리, 바코드, 브랜드, 상품명, MOQ, LT, 중국재고(쿠팡/일반), 사방넷재고, FC재고, 미입고(쿠팡/일반), 재고합계, 물류실재고, 재고소진일, 안전재고일수, 입고예정코드, 입고일자, 쿠팡발주수량, 주간판매, 예상판매6주, 발주량산출, 재고상태
- 재고상태: 위험(발주량 음수) / 주의(발주량 < 예상판매) / 정상
- 필터·검색·정렬, 체크박스 선택 후 발주 조회 연동
- 물류팀 뷰: 실재고 입력 후 재고합계 실시간 재계산
- 오래된 입고일자 확인 모달 (과거 날짜 행 처리)
- **100행씩 페이지네이션** (렌더링 부하 경감)

## 구글시트 발주요청리스트 컬럼 구조 (고정)
시트명: `[엔대시]중국 발주요청리스트(交期管理表)`

| 열 | 인덱스 | 내용 |
|----|--------|------|
| A  | 0      | 쿠팡 중국재고 |
| B  | 1      | 일반채널 중국재고 |
| C  | 2      | 발주일자 |
| D  | 3      | 바코드 |
| E  | 4      | 발주번호 |
| F  | 5      | 쿠팡 미입고(발주수량) |
| G  | 6      | 일반 미입고(발주수량) |
| H  | 7      | 제품명 |
| I  | 8      | 비고/입고예정일 |
| J~L| 9~11  | 중국직원 사용 (중국출발일, 누아트입고일자, 14일경과) |
| M  | 12     | 제조사 |

- 중국재고: 빈 셀 무시, 마지막 비어있지 않은 값 사용
- 미입고: 바코드별 누적 합산
- 숫자에 콤마 포함 가능 (`3,000`, `9,999`) → 정상 처리

## SMTP 이메일 설정
- 서버: smtp.naver.com / 포트: 465 (SSL)
- .env 파일에 SMTP_FROM, SMTP_PASSWORD(앱 비밀번호), SMTP_TO, SMTP_CC 설정
- ehlo("localhost") 필수 (Windows 한글 호스트명 오류 우회)
